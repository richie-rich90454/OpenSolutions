# Security Policy

## Reporting a Vulnerability

Do not open a public issue for security vulnerabilities.

Instead, email **security@lernchih.dev** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

You should receive a response within 48 hours.

## Disclosure Policy

- Vulnerabilities are disclosed after a fix is released
- We coordinate with reporters on disclosure timing
- Credit is given to reporters unless they prefer to remain anonymous

## Supported Versions

Only the latest release receives security updates.

## Scope

The following are in scope:
- Authentication bypass
- SQL injection
- XSS
- CSRF
- Authorization flaws
- Sensitive data exposure

The following are out of scope:
- Denial of service
- Social engineering
- Third-party service vulnerabilities
- Issues in dependencies (report to the upstream project)
